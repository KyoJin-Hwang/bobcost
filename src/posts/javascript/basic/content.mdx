## 💡Javascript 기초 정리하게 된 계기
React와 같은 프레임워크를 사용하면서 점점 JavaScript의 기초 개념을 놓치고 있다는 생각이 들었습니다. 
실제로 간단한 로직을 작성할 때도 문법이나 동작 방식이 헷갈릴 때가 많았고, 기초를 탄탄히 다지지 않으면 실력이 더 이상 올라가지 않을 거란 위기감이 들었습니다. 그래서 다시 한 번 JavaScript의 기본 개념들을 정리하며 제대로 이해하고 넘어가자는 마음으로 이 글을 작성하게 되었습니다.

## 📘 JavaScript란?

**JavaScript**는 웹 브라우저에서 실행되는 대표적인 프로그래밍 언어로, 웹 페이지를 **동적으로 조작**하고, **사용자와의 상호작용**을 가능하게 합니다.

> ✅ **HTML**은 웹 페이지의 **구조(structure)** 를 만들고,  
> ✅ **CSS**는 그 구조에 **스타일(style)** 을 입히며,  
> ✅ **JavaScript**는 웹 페이지에 **행동(behavior)** 을 부여합니다.

---

### 🧾 정의
- **객체 기반(Object-oriented)**  
- **이벤트 기반(Event-driven)**  
- **동적 타입(Dynamically typed)**  
- **인터프리터 언어(Interpreted language)**
> 브라우저가 HTML과 CSS를 해석한 후, JavaScript는 DOM(Document Object Model)을 제어하여 페이지에 생명력을 불어넣습니다.
---

### 🕘 역사 간단 정리
- **1995년**: Netscape의 브렌던 아이크(Brendan Eich)가 10일 만에 JavaScript의 초안 개발  
- **초기 이름**은 Mocha → LiveScript → JavaScript로 변경  
- Java와는 이름만 비슷할 뿐, 전혀 다른 언어입니다  
- 이후 ECMAScript(표준 스펙)로 발전하여 지속적으로 업그레이드됨
---

### 🧠 JavaScript는 어떤 역할을 하나요?
JavaScript는 웹 페이지에 다음과 같은 **동적 기능**을 제공합니다:
- 버튼 클릭 시 UI 변경  
- 입력값 검증  
- 모달 창 열기/닫기  
- 슬라이더, 드롭다운, 애니메이션 처리  
- 서버와의 데이터 통신 (AJAX, fetch 등)  
- 실시간 채팅, 알림, 비디오 플레이어 제어 등

## ⭐️ 자료형이란?
자바스크립트에서 값은 항상 **문자열**이나 **숫자형** 같은 특정한 자료형에 속합니다.

자바스크립트에는 여덟 가지의 기본 자료형이 있습니다.

자바스크립트의 변수는 자료형에 관계없이 모든 데이터일 수 있습니다. 따라서 변수는 어떤 순간에 문자열일 수 있고 다른 순간엔 숫자가 될 수도 있습니다.

```js showLineNumbers ${1-2}
let message = '안녕 나는 문자열이야'
message = 12345 

console.log(message) 
// 👉 결과: 12345
```
> 이처럼 자료의 타입은 있지만 변수에 저장되는 값의 타입은 언제든지 바꿀 수 있는 언어를 ‘동적 타입(dynamically typed)’ 언어라고 부릅니다.

### 📌 원시 자료형 (Primitive Type)
- 객체가 아니며, 메서드를 가지지 않습니다.  
- 하나의 값만을 저장합니다.  
- 변수에 값 자체가 저장되며, 복사 시 값이 그대로 복제됩니다.  
- 변경이 불가능한(immutable) 값입니다.  

#### 종류(원시)
- `string` : 문자열  
- `number` : 숫자 (정수 및 실수)  
- `boolean` : true 또는 false  
- `null` : 의도적으로 "값이 없음"을 명시  
- `undefined` : 값이 할당되지 않은 변수  
- `symbol` : 고유하고 변경 불가능한 값  
- `bigint` : 아주 큰 정수를 다룰 수 있는 자료형

### 📌 참조 자료형 (Reference Type)
- 객체를 기반으로 하며, 여러 개의 값을 담을 수 있는 구조입니다.  
- 변수에는 값이 아닌 해당 객체의 참조 주소가 저장됩니다.  
- 복사 시 참조값이 복제되어, 두 변수는 같은 객체를 가리키게 됩니다.  
- 변경 가능한(mutable) 데이터입니다.  

#### 종류(참조)
- `object` : 키-값 쌍으로 구성된 데이터 집합  
- `array` : 순서가 있는 값들의 리스트  
- `function` : 재사용 가능한 코드 블록이며, 일급 객체로 취급됨  
- `date` : 날짜와 시간 데이터를 표현하기 위한 객체  
- `regexp` : 정규표현식 처리를 위한 객체  
- `map` : 키-값 쌍의 컬렉션으로, 키에 어떤 타입이든 사용 가능  
- `set` : 중복되지 않는 유일한 값들의 집합

## 🚀 자료형

### 📌 숫자형 (Number)

- 숫자형(number type) 은 정수 및 부동소수점 숫자(floating point number)를 나타냅니다.
- 숫자형과 관련된 연산은 다양한데, 곱셈 *, 나눗셈 /, 덧셈 +, 뺄셈 - 등이 대표적입니다.
- 숫자형엔 일반적인 숫자 외에 Infinity, -Infinity, NaN같은 '특수 숫자 값(special numeric value)'이 포함됩니다.

```js showLineNumbers
let num = 123456
num = 12.3456
```

**NaN**은 계산 중에 에러가 발생했다는 것을 나타내주는 값입니다. 부정확하거나 정의되지 않은 수학 연산을 사용하면 계산 중에 에러가 발생하는데, 이때 NaN이 반환됩니다.

| 표현식               | 결과            | 이유                                                                 |
|--------------------|----------------|----------------------------------------------------------------------|
| "문자열" ( / , * , - ) 숫자   | `NaN`          | `/`, `*`, `-` 는 숫자 연산을 시도하지만 `"문자열"`은 숫자로 변환할 수 없기 때문에 `NaN` 반환 |
| "문자열" + 숫자   | `"문자열숫자"` | `+`는 문자열 연결 연산자처럼 작동하여 숫자를 문자열로 변환 후 연결함          |

```js showLineNumbers
console.log("숫자가 아님" / 2);
console.log("숫자가 아님" * 2);
console.log("숫자가 아님" - 2);
// 👉 결과: NaN
// 👉 결과: NaN
// 👉 결과: NaN
console.log("숫자가 아님" + 2); 
// 👉 결과: "숫자가 아님2"
```

### 📌 문자형 (String)
자바스크립트에선 문자열(string)을 따옴표로 묶습니다.

문자열을 나타내는 방식은 3가지가 있습니다.
- 큰따옴표: "Hello"
- 작은따옴표: 'Hello'
- 역 따옴표(백틱, backtick): `` `Hello` ``

따옴표와 작은따옴표는 ‘기본적인’ 따옴표로, 자바스크립트에서는 이 둘에 차이를 두지 않습니다.
역 따옴표로 변수나 표현식을 감싼 후 `${…}`안에 넣어주면, 아래와 같이 원하는 변수나 표현식을 문자열 중간에 손쉽게 넣을 수 있습니다.

```js showLineNumbers
let str = "Hello!";
let str2 = 'Hello 2!';
let str3 = `Owen ${str}`;
```

--- 

### 📌 불린형 (Boolean)
불린형(논리 타입)은 true와 false 두 가지 값밖에 없는 자료형입니다.
불린형은 긍정(yes)이나 부정(no)을 나타내는 값을 저장할 때 사용합니다. true는 긍정, false는 부정을 의미합니다.

```js showLineNumbers
let yes = true
let no = false

let isGreater = 4 > 3
console.log(isGreater)
// 👉 결과: true
```

---

### 📌 null 값 (null)
`null` 값은 지금까지 소개한 자료형 중 **어느 자료형에도 속하지 않는 값**입니다.
`null` 값은 오로지 `null` 값만 포함하는 **별도의 자료형**을 만듭니다.


```js showLineNumbers
// let age = null; 은 나이(age)를 알 수 없거나 그 값이 비어있음을 보여줍니다.
let age = null;
```

자바스크립트에선 null 사용하는 방법
- ‘존재하지 않는(nothing)’ 값
- ‘비어 있는(empty)’ 값
- ‘알 수 없는(unknown)’ 값을 나타내는 데 사용합니다. 

### 📌 undefined 값 (undefined)
`undefined` 값도 `null` 값처럼 자신만의 자료형을 형성합니다.
`undefined`는 '값이 할당되지 않은 상태’를 나타낼 때 사용합니다.
변수는 선언했지만, 값을 할당하지 않았다면 해당 변수에 `undefined`가 자동으로 할당됩니다.
